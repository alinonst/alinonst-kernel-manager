# 贡献指南

感谢您对 `alinonst-kernel-manager` 的兴趣！我们欢迎任何形式的贡献。

## 提交 Issue

- 使用 Issue 模板描述问题或建议。
- 请提供详细的系统信息（Arch Linux 版本、内核版本、错误日志等）。

## 提交 Pull Request

1. Fork 本项目。
2. 创建您的功能分支 (`git checkout -b feature/amazing-feature`)。
3. 提交您的更改 (`git commit -m 'Add some amazing feature'`)。
4. 推送到分支 (`git push origin feature/amazing-feature`)。
5. 打开 Pull Request。

## 代码风格

- 使用 C++17 标准。
- 遵循项目现有代码风格（缩进 4 空格，使用 `snake_case` 命名函数和变量）。
- 添加适当的注释，特别是复杂逻辑部分。

## 测试

确保您的更改在本地通过编译，并测试所有功能（安装、配置仓库、CPU 检测等）。强烈建议在干净虚拟机或容器中测试。
